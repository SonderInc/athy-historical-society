# Agents Guide – Athy Historical Society

## Project Architecture

This is a TanStack Start (React SSR) app deployed on Netlify. It has a single public-facing page and three API endpoints backed by a Netlify Postgres database.

## Key Directories

```
src/
  routes/
    __root.tsx          – HTML shell, meta tags
    index.tsx           – Main page (content + sign-up form + participants list)
    verify.tsx          – Email verification landing page
    api/
      submit.ts         – POST /api/submit — saves participant, sends verification email
      verify.ts         – GET  /api/verify  — verifies token, marks participant confirmed
      participants.ts   – GET  /api/participants — returns verified names (JSON)
  server/
    email.server.ts     – Nodemailer helper (server-only, never import on client)

db/
  schema.ts             – Drizzle schema (participants table)
  index.ts              – Drizzle client initialised with Netlify DB adapter

netlify/database/migrations/
                        – Auto-generated SQL migrations (apply automatically on deploy)

drizzle.config.ts       – Drizzle Kit config (output → netlify/database/migrations)
```

## Coding Conventions

- API routes live in `src/routes/api/` and use the `server.handlers` pattern from TanStack Start.
- Server-only helpers (DB access, email) live in `src/server/` with a `.server.ts` suffix — never import these in client components.
- All imports from `db/` use the `.js` extension (required for ESM bundler resolution).
- Environment variables are read via `process.env.*` inside server-only code.

## Required Environment Variables

| Variable | Purpose |
|---|---|
| `SMTP_HOST` | Outbound SMTP server hostname |
| `SMTP_PORT` | SMTP port (default 587) |
| `SMTP_USER` | SMTP username |
| `SMTP_PASS` | SMTP password |
| `FROM_EMAIL` | Sender address for verification emails |
| `SITE_URL` | Public base URL (for building verification links) |

## Non-obvious Decisions

- Email verification uses a random 32-byte hex token stored in the `participants` table. The token is not invalidated after verification — duplicates are caught by the `verified` flag.
- The `/api/verify` endpoint issues a redirect to the homepage with a query param (`?verified=success|invalid|already`) so the main page can show a contextual banner.
- Only the participant's **name** is returned by `/api/participants` — the email address is never exposed via any public endpoint.
- The `routeTree.gen.ts` file is auto-generated by TanStack Router's Vite plugin at build time — do not edit it manually.
